# AI Usage Guidelines



## AI Tools Commonly Used in Software Engineering



Some AI tools commonly used in software engineering and development roles include:



* ChatGPT

* GitHub Copilot

* Claude

* Gemini

* Cursor AI

* Perplexity AI



These tools can assist with coding, debugging, documentation, learning new technologies, brainstorming solutions, and improving productivity.



---



# Benefits and Risks of Using AI



## Benefits



* Faster research and learning.

* Assistance with coding and debugging.

* Improved productivity.

* Help with documentation and writing.

* Brainstorming ideas and alternative solutions.

* Simplifying complex concepts.



## Risks



* Incorrect or outdated information.

* Security and privacy risks if sensitive data is shared.

* Over-reliance on AI-generated solutions.

* Reduced critical thinking and problem-solving skills.

* AI-generated code may contain bugs or security vulnerabilities.



---



# Information That Should Never Be Entered Into AI Tools



The following information should never be entered into public AI tools unless explicitly approved:



* Company confidential information.

* Customer or user data.

* Passwords and API keys.

* Authentication tokens.

* Private source code repositories.

* Financial information.

* Personal information about customers, employees, or users.

* Internal business documents and strategies.



---



# How to Validate AI-Generated Content



AI-generated content should always be reviewed before use.



Methods include:



* Checking official documentation.

* Comparing information with trusted sources.

* Testing AI-generated code before deployment.

* Verifying technical claims independently.

* Reviewing outputs for logical errors and inaccuracies.



AI should be treated as an assistant rather than an authority.

# Reflection



## When Should You Use AI for Assistance and When Should You Rely on Your Own Skills?



AI is most useful for:



* Learning unfamiliar concepts.

* Research and brainstorming.

* Generating code examples.

* Debugging assistance.

* Drafting documentation.



I should rely on my own skills when:



* Making technical decisions.

* Reviewing code quality.

* Solving problems independently first.

* Validating the correctness of AI-generated output.

* Working with sensitive company information.



---



## How Can You Avoid Over-Reliance on AI While Still Benefiting From It?



To avoid over-reliance on AI:



* Attempt problems independently before asking AI for help.

* Use AI as a learning tool rather than a replacement for thinking.

* Understand AI-generated solutions before using them.

* Verify outputs through testing and research.

* Continue practicing problem-solving without AI assistance.



---



## What Steps Will You Take to Ensure Data Privacy When Using AI Tools?



I will:



* Never upload confidential company information.

* Avoid sharing customer or user data.

* Remove sensitive information from prompts.

* Follow company policies regarding AI usage.

* Verify that any AI tool used is approved for the intended purpose.

# Task



## One Task Improved Using AI



One task I improved using an AI tool was learning and applying GitHub workflow skills during onboarding.



I used ChatGPT to understand Git commands, repository cloning, remotes, committing, pushing changes, and troubleshooting errors in Git Bash. This helped me complete onboarding tasks more confidently and understand what each command was doing instead of copying commands without context.



Examples included:



- Understanding the difference between `origin` and `upstream`

- Fixing Git Bash path errors

- Creating and committing Markdown files

- Pushing changes to my GitHub repository

- Troubleshooting when changes were not staged before committing



## Critical Review of the Output



The AI-generated guidance required careful checking before use. I verified commands by reading the terminal output, checking `git status`, and confirming changes on GitHub after pushing.



This showed me that AI can be very helpful for learning technical workflows, but I still need to understand each step, check the results, and avoid blindly running commands.
